# doc-writer | 算法工程师受控文档引擎

`doc-writer` 是一套专为算法工程师设计的技术文档产出工具集。它通过将技术素材（代码、数据、文献）转化为 HTML 物理隔离的章节，并最终通过 MathML 固化与样式映射技术，生成 100% 对标理 **官方 Word 受控模板** 的正式报告。

## 🌟 核心特性

1. **样式全自动映射**：无需手动调整格式。系统自动将 HTML 语义标签（H1-H4, P, Table）映射至模板内置的“文章一级标题”等受控样式。
2. **原生 MathML 公式转换**：采用 `latex2mathml` 固化技术，导出的 Word 公式为原生可编辑的 OMML 算式，告别图片公式或 OLE 对象。
3. **参考文献自动化**：内置 `extract_references.py`，自动扫描 `src/docs/` 下的 Markdown 文献题录并生成带索引的参考文献章节。
4. **标题编号智能冲突处理**：内置正则剥离引擎，自动清除 HTML 标题中的硬编码序号（如 `3.1. `），完美适配 Word 模板的自动多级列表功能。
5. **章节物理文件隔离**：采用多 HTML 文件组织形式，支持章节级锁定与协作，规约误操作风险。

## 🚀 4 阶段标准工作流

### 1. 确认路线 (Planning)
定义目标路径并准备素材（参考文献/数据图表）。

### 2. 生成 HTML (Development)
根据素材生成独立章节 HTML。若有参考文献，运行脚本：
```bash
python scripts/extract_references.py <项目路径>
```

### 3. 二次评审 (Review)
在浏览器预览 `index.html`。对特定章节进行修改，并在 `task.md` 中标记锁定。

### 4. 最终导出 (Export)
执行全链路导出流程：
1. **合并 HTML**: 将所有章节合成分离为 `merged.html`。
2. **公式固化**: 运行 `python scripts/latex_to_mathml.py <merged.html路径> <output.html路径>`。
3. **Pandoc 转换**:
   ```bash
   pandoc merged_mathml.html -f html -t docx --mathml --reference-doc="references/EDAN_word模板.docx" -o "output.docx"
   ```
4. **样式重映射**:
   ```bash
   python scripts/apply_docx_format.py "output.docx"
   ```
